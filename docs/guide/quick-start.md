# DKU-小X宝社区 创新校园项目：Quick Start & Must-Know Guideline

## 项目背景与目标

- **发起单位**：由 DKU、CMC、小X宝社区与相关公益基金会联合发起。
- **项目方向**：面向肿瘤/罕见病患者及其家庭的公益项目。
- **参与形式**：面向校园招募约 6 组学生团队，通过真实问题驱动的项目实践，探索科技与公益结合的创新方案。
- **核心目标**：
  - **帮助真实个体**：哪怕只真正帮助到 1 位患者/家庭，这个项目就已经值得。
  - **培养正确的公益观**：倡导长期陪伴、多方协作、尊重专业与合规。
  - **提升同学的综合能力**：包括同理心、项目管理、产品/技术能力与跨学科协作能力。

## 3 条基准提醒

为了让项目能够顺利展开，我们准备了这个指南，目的是：

- **告知**：说明你需要注意和遵守的执行原则。请尽量遵从这些原则，有助于社区资源获取、志愿者沟通、项目推进，建立正确的公益观，实现本次项目目标。
- **保护和监管**：本次项目可能涉及与肿瘤或罕见病患者/家庭的交流，请特别注意沟通方式以及自我心理防护。所有项目和沟通过程，均应接受 DKU / CMC / 基金会的指导和透明化监督。**社区不鼓励**单独 1v1 的私下沟通，鼓励在可行范围内保持沟通透明、信息同步。
- **合规**：本次项目可能涉及数据合规和隐私保护，必须遵从国家法律和社区约定，严格禁止任何形式的数据违规行为，确保伦理合规。

## 管理架构与角色分工

### 监督委员会（Project Management Committee，PMC）

- **构成**：由 DKU / CMC / 基金会代表组成。
- **职责**：
  - **统筹与监督**：保证项目符合公益初衷与伦理法规要求。
  - **资源协调**：在合理范围内为同学提供必要的资源与支持。
  - **风险把控**：对涉及数据、合规、合作方等重大事项进行把关。

### 项目管理角色

- **PMC**：进行整体统筹管控，解决跨项目的资源–人力–进度的统一协调。
- **Mentor（导师）**：按项目匹配，为对应项目组提供指引与支持，包括：
  - 项目方向与范围的把握
  - 技术或产品上的关键建议
  - 资源对接与过程陪伴
- **Leader（项目负责人）**：
  - 负责本项目组的整体实现，包括团队组织、进度设计、产品定义、产品实现、用户测试与反馈，以及文档记录。
  - 在项目内部承担最终协调与决策责任（在合规框架内）。
- **Member（项目成员）**：
  - 负责具体任务的实现（调研、设计、开发、测试、运营等）。
  - 对自己认领的任务负责，及时同步进展和困难。

> **建议**：每个项目组尽量明确 Leader 和备选 Leader，建立最基本的项目管理节奏（例如每周一次站会）。

## 沟通与协作机制

- **推荐沟通方式**：
  - **项目微信群**：透明沟通，让监督委员会（DKU / CMC / 基金会代表）、 Mentor 了解你的付出、进展、困难和反馈，也更容易获得支持。
  - **GitHub 仓库**：鼓励加入专用仓库，为自己项目建立目录（或独立仓库），在 issue 中求助，让开源世界来帮助你。
  - **线上/线下会议**：重要决策建议在多人会议中完成，并简要记录纪要。
- **需避免的情况**：
  - 尽量避免绕开监督委员会的私下深度沟通，尤其是涉及资金、数据、患者隐私、商业合作等敏感议题。
  - 对于不在官方指导和记录范围内的沟通，社区无法进行管理，相应风险和责任可能由你个人承担。

> **小提示**：遇到让你不舒服、拿不准是否合规或不知如何拒绝的请求时，优先在群内或单独向 Mentor / DKU 老师求助，而不是一个人扛。

## 项目周期与关键里程碑（示例）

不同项目的具体节奏会有差异，但一般可以参考下面的阶段性安排（以若干月为例，仅供你在组内做计划时参考）：

- **阶段一：理解与聚焦（约 1–2 周）**
  - **输出**：项目一页纸（项目简介、目标用户、要解决的核心问题、预期影响）。
- **阶段二：方案设计与原型（约 2–4 周）**
  - **输出**：产品/服务原型（可以是线框图、流程图、MVP、小工具 Demo 等），以及简要用户旅程说明。
- **阶段三：实现与迭代（约 4–6 周）**
  - **输出**：可运行版本 / 实施方案，收集初步反馈（如小范围用户测试、专家反馈、导师建议）。
- **阶段四：总结与分享**
  - **输出**：项目总结报告、展示材料（slides / demo）、后续可持续建议。

> **重要共识**：本项目不以“商业成功”作为衡量标准，而以“是否真正尊重个体、是否认真尝试解决一个真实问题”为核心评价。

## 与患者/家庭沟通的注意事项

- **你不是医生**：
  - 不提供诊断建议、不推荐具体药物或治疗方案。
  - 避免给出“保证有效”“一定没问题”之类的承诺。
- **尊重与共情**：
  - 允许对方表达真实情绪（焦虑、愤怒、悲伤等），但不试图“快速解决一切问题”。
  - 使用平等、尊重的语气，避免居高临下或“替他们决定”。
- **适度的边界感**：
  - 可以倾听、提供信息和工具，但不要过度卷入对方家庭私人决策。
  - 如感到压力过大，可以及时向 Mentor 或 DKU 老师求助。
- **引导专业资源**：
  - 对于医疗上的关键问题，鼓励对方回到主诊医生或正规医疗机构。
  - 如项目中涉及专业信息表达，尽量让专业人士做审核把关。

## 数据、合规与伦理要求（必读）

- **什么是敏感/隐私数据？**
  - 个人身份信息：姓名、身份证号、联系方式、住址、工作单位、具体就诊医院与科室等。
  - 健康与病情信息：具体疾病诊断、治疗方案、检查结果、用药情况等。
  - 其他可以单独或组合识别到某一个人的信息。
- **你必须做到**：
  - **不主动收集**任何与项目无关的个人隐私数据。
  - 如确有必要收集（例如用户测试、问卷等），必须：
    - 事先经过 DKU / CMC / 基金会及相关伦理/审批流程同意；
    - 明确告知对方用途、保存方式和使用边界；
    - 尽可能进行脱敏和匿名化处理。
  - 不在非安全环境（如个人云盘、个人公开仓库等）存放含有个人隐私的数据。
- **严禁行为**：
  - 未经授权采集、保存、传播任何患者或家庭的个人隐私数据。
  - 将项目中接触到的个人信息用于与项目无关的目的（比如商业化尝试、个人研究等）。
  - 诱导对方提供隐私信息，尤其是以“帮忙”“换取好处”为理由。
- **违规后果**：
  - 触碰隐私与数据安全红线，可能会导致：
    - 在社区层面的撤项、退出项目、公开通报与谴责；
    - 在学校或合作机构层面的纪律处理；
    - 在严重情况下，依据中国相关法律法规承担法律责任。

## Quick Start 清单（给新加入的你）

- **24 小时内**：
  - 加入项目微信群和指定 GitHub 组织/仓库。
  - 加入小x宝社区志愿者，实名登记，开通飞书知识库 (https://info.xiao-x-bao.com.cn)
  - 仔细阅读本《Quick Start & Must-Know Guideline》。
- **1 周内**：
  - 完成团队内部自我介绍，明确 Leader 和暂定分工。
  - 与 Mentor 做一次项目期望对齐（线上或线下）。
  - 产出项目一页纸（项目简介、问题、用户、初步想法）。
- **2–3 周内**：
  - 梳理出第一版可展示成果（例如流程图、原型图、简单 Demo、调研报告）。
  - 明确接下来 4–6 周的目标与关键节点。
- **行为准则 Checklist**：
  - **尊重**：尊重患者、家属、合作方与组员。
  - **透明**：重要沟通尽量在可记录和可见的渠道中进行。
  - **守时**：尊重大家的时间，按时参加会议和交付任务。
  - **反馈**：遇到问题及时说，遇到不适及时求助。

## 常见问答（FAQ）

- **Q：“我怕这个项目做失败了，该怎么考虑？”**  
  **A**：记住，我们的成功不是以商业价值判断。如果你没有放弃，认真完成了一个应用/方案，并且哪怕只帮助到一位患者或家属，我们就认为这个项目具有价值。一个人的生命和一百个人的生命，在尊严和意义上是等价的。

- **Q：“如何沟通和反馈、求助和推动问题解决？”**  
  **A**：推荐流程：
  - **优先在项目微信群沟通**：透明沟通，让监督委员会和 Mentor 理解你的进展、困难与需求，便于提供支持。
  - **善用 GitHub issue**：在仓库中提出清晰的问题或需求，便于他人异步协作。
  - **必要时约时间深聊**：与 Mentor 或 PMC 成员安排 1:1 或小范围讨论，但重要结论建议在群内做简要同步。

- **Q：“谁来指导？”**  
  **A**：社区会根据项目，为每个项目匹配一位 Mentor（导师），提供项目指引、开发辅助、资源协调以及疑难问题解答。视情况也可能邀请领域专家参与阶段性点评或短期指导。

- **Q：“如果我想放弃和退出，可以吗？”**  
  **A**：我们理解每个人都有客观情况和压力，但希望你在“退出”之前先尝试：
  - **调整节奏**：合理拉长时间、降低阶段目标，不要把自己压得太紧。这里不是商业公司，我们没有严格的商业化 deadline。
  - **多沟通**：孤独是很多项目中断的原因之一。试着在组内、与 Mentor、与同学多聊聊，你会发现很多“攀登者”在路上。
  - **善用工具**：不要重复造轮子，GitHub 和开源世界中已有大量成果可以参考和复用。
  如果最终你仍决定退出，也没有问题，请联系 DKU 老师或 Mentor，完成必要的文档记录和交接，我们期待未来在别的时刻再合作一次。

- **Q：“我参加的是什么项目？”**  
  **A**：本次计划会列出 5–6 个围绕肿瘤/罕见病公益方向的项目主题。在报名和沟通中，你提交的项目意向会被优先考虑；我们会综合团队情况、导师配比和项目成熟度来做最终分配。

- **Q：“如果我不满意我的项目或导师，怎么办？”**  
  **A**：可以向监督委员会的任一代表反馈（DKU 老师通常是更直接的入口），说明你的感受与诉求。我们鼓励用建设性、具体的方式表达不满（例如指出具体场景、期待的改进），共同寻找更好的安排，而不是憋在心里。

- **Q：“如果过程中，我被要求为参与项目无关的事情投入时间或金钱，或者觉得对方行为涉嫌数据违规或非公益化商业目的，我该如何处置？”**  
  **A**：请你立即提高警惕，并：
  - **第一时间向 DKU / CMC / 基金会成员反馈**，保留聊天记录、邮件等证据。
  - **鼓励你坚定拒绝不当做法和不合理要求**，包括但不限于：与项目无关的付费购买、兼职/劳务安排、过度索要隐私信息、要求放弃知识产权等。
  社区在项目内外都**严禁**借公益名义进行任何不当交易、商业化引导或隐私数据获取。一旦触碰红线，将面临社区顶格处理、学校或机构层面的严肃追责，严重者也可能触犯法律。

- **Q：“我需要注意哪些合规要求，社区有什么建议？”**  
  **A**：可以按下面三个角度来记：
  - **法律合规**：遵守中国关于个人信息保护、数据安全和互联网内容的相关法律法规，宁愿“多问一句”，也不要“想当然”。
  - **机构规范**：尊重 DKU / CMC / 基金会的内部规范和导师建议，涉及数据、合作方、对外宣传时，提前报备和沟通。
  - **个人底线**：对任何让你感到不适、模糊或“不太对劲”的要求，保持警惕并主动求助。宁可慢一点，也不要冒险踩红线。
