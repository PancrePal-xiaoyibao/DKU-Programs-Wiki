## 1. RAG 产品技术栈工具

### 什么是 RAG？
RAG（Retrieval-Augmented Generation，检索增强生成）是一种让 AI 能够"查阅"特定文档后再回答问题的技术。简单说：你可以把医疗文档、患者手册、治疗指南等上传到系统，让 AI 基于这些内容来回答用户提问，而不是胡编乱造。

**对你的价值**：如果你的项目需要为患者提供疾病知识问答、用药指南查询、就医流程咨询等功能，RAG 工具可以帮你快速搭建一个"专属知识库 + AI 客服"，无需从零写代码。



### 社区推荐（小白友好）

#### **Fastgpt**
- **官网**：[www.fastgpt.io](https://www.fastgpt.io)
- **在线体验**：[https://cloud.fastgpt.io/](https://cloud.fastgpt.io/)
- **为什么推荐给小白**：
  - 支持中英双语界面，上手门槛低。
  - 提供官方 SaaS 版本，注册后可以直接在浏览器里拖拽式搭建知识库问答系统，不需要自己部署服务器。
  - 适合快速验证想法：比如你想做一个"骨肉瘤患者常见问题解答助手"，可以先把相关文档上传到 Fastgpt，测试效果，再决定是否深度开发。
- **如何上手**：
  1. 访问 [https://cloud.fastgpt.io/](https://cloud.fastgpt.io/) 注册账号。
  2. 创建一个"知识库"，上传你的文档（支持 PDF、Word、Markdown 等）。
  3. 创建一个"应用"，关联知识库，简单配置后即可对外提供问答服务。
  4. 可以生成分享链接或嵌入到你的网页/小程序中。

#### **Dify**
- **官网**：[https://dify.ai/](https://dify.ai/)
- **为什么推荐**：
  - 开源且功能强大，支持多种 LLM（大语言模型）接入。
  - 提供可视化工作流编排，适合需要更复杂逻辑的场景（例如：先查知识库，再调用外部 API，最后生成个性化回复）。
  - 社区活跃，文档相对完善，遇到问题容易找到解决方案。
- **适用场景**：如果你的项目需要"知识问答 + 外部数据调用"（比如查询药品数据库、医院信息），Dify 会是更灵活的选择。
- **如何上手**：
  1. 访问官网了解功能，可以先用官方 Demo 体验。
  2. 如果需要自己部署，可以参考 GitHub 上的部署文档（需要一定的服务器和 Docker 基础，建议找 Mentor 或技术同学协助）。
  3. 配置知识库、选择 LLM、设计对话流程，测试后发布。



### 鼓励探索（进阶）
如果你已经熟悉基础 RAG 工具，想要尝试更前沿的技术，可以关注：

- **Ragflow**：基于图数据库的 RAG，适合处理复杂关系数据（例如：疾病-症状-药物之间的关联）。
- **多智能体协同驱动的 RAG**：让多个 AI 协作完成任务（例如：一个负责查文档，一个负责总结，一个负责生成建议）。
- **技术前沿项目**：
  - **TrustRAG**：注重可信度和可解释性的 RAG 系统，适合医疗等对准确性要求高的场景。

# 2. AI 编程助手（Coding Copilot）

### 什么是 AI 编程助手？
AI 编程助手是能够帮你写代码、解释代码、修复 Bug、甚至根据自然语言描述直接生成完整功能的工具。你可以把它理解为"一个 24 小时在线的编程导师 + 结对编程伙伴"。

**对你的价值**：
- **编程小白**：可以用自然语言描述需求（例如："帮我写一个网页表单，收集用户姓名和联系方式"），AI 会直接生成代码，并解释每一行的作用。
- **有一定基础的同学**：可以用 AI 加速开发，比如快速搭建项目框架、自动补全重复代码、帮你调试报错信息。
- **项目中的实际应用**：无论是做小程序、网页、数据处理脚本，AI 编程助手都能大幅降低开发门槛。



### 黑客松指定合作产品（优先推荐）

#### **Qoder**
- **官网**：[https://qoder.com](https://qoder.com)
- **为什么推荐**：
  - **本次项目的官方合作工具**，社区成员使用 Qoder 可以获得额外支持和资源。
  - 支持中文对话，适合国内同学。
  - 提供完整的 IDE（集成开发环境）+ AI 辅助，可以在线编写、运行、部署代码。
- **适用场景**：适合从零开始的项目，尤其是需要快速原型验证的小工具、网页应用。
- **如何上手**：
  1. 访问 [https://qoder.com](https://qoder.com) 注册账号。
  2. 创建一个新项目，选择你需要的技术栈（如 React、Python 等）。
  3. 在对话框中用自然语言描述需求，AI 会帮你生成代码和项目结构。
  4. 在线调试和预览，满意后可以部署或导出代码。

#### **Weavefox**
- **为什么推荐**：也是黑客松合作产品之一，功能类似，可以作为备选方案。



### 其他主流 AI 编程工具（可选）

#### **Cursor**
- **特点**：基于 VSCode 的 AI 编程工具，支持代码补全、对话式编程、多文件编辑。
- **适合人群**：有一定编程基础，习惯使用专业 IDE 的同学。
- **如何获取**：访问 [https://cursor.sh](https://cursor.sh) 下载客户端。

#### **Claude Code**（Anthropic 出品）
- **官网**：[https://claude.com/product/claude-code](https://claude.com/product/claude-code)
- **特点**：基于 Claude 大模型，擅长代码解释和安全性审查。
- **适合场景**：如果你需要理解一段复杂代码，或者希望 AI 帮你检查代码中的潜在风险（例如数据泄露、SQL 注入），Claude Code 会是不错的选择。

#### **Windsurf**、**OpenCode**、**Amp**、**Antigravity**、**Trae**
- 这些工具各有特色，功能上与 Cursor 类似，可以根据个人偏好选择。建议优先使用黑客松合作产品（Qoder），遇到瓶颈时再尝试其他工具。



### 小白使用建议
- **第一步**：从 Qoder 开始，因为它有中文支持且与本项目深度合作。
- **遇到问题时**：直接在对话框里问 AI"这段代码是什么意思？""报错了怎么办？"，AI 会给你解释和修复建议。
- **进阶技巧**：学会"提示词工程"——描述需求时尽量具体（例如："用 React 写一个表单，包括姓名、年龄、联系方式三个字段，提交后在控制台打印数据"），AI 给出的代码质量会更高。  

# 3. LLM Token 资源（AI 调用额度）

### 什么是 LLM Token？
当你使用 AI 服务（如 GPT、Claude、GLM 等）时，每次对话、每次生成内容都会消耗"Token"（可以理解为"字数额度"）。Token 通常需要付费购买，但作为学生和公益项目参与者,你可以通过以下渠道获取免费或优惠资源。

**对你的价值**：
- 免费获取 AI 调用额度，用于项目开发和测试。
- 降低成本，让你可以放心尝试各种 AI 功能，而不用担心"用多了要花钱"。



### 3.1 社区资源（优先推荐）

#### **向 Mentor 申请社区配额**
- **如何申请**：联系你的项目 Mentor 或田老师，说明你的项目需求和预计使用量。
- **为什么优先推荐**：
  - 社区配额通常额度更大，适合生产环境使用（例如：你的应用正式上线后，真实用户会持续调用 AI）。
  - 由社区统一管理，可以避免个人账号被滥用或超额的风险。
- **注意事项**：
  - 申请时请说明用途（例如："用于 RAG 知识库问答"或"用于 AI 编程助手"），方便 Mentor 评估和分配。
  - 珍惜资源，不要用于与项目无关的个人娱乐或商业用途。



### 3.2 免费资源（自助获取）

# 4. 自动化工作流工具

### 什么是自动化工作流？
自动化工作流工具可以帮你把多个步骤串联起来，实现"无需人工干预的自动化任务"。例如：
- 当用户在表单提交问题后，自动发送到 AI 处理，并把回复结果通过邮件发送给用户。
- 定时从数据库中提取数据，生成报表并发送到微信群。
- 监控某个网站的更新（例如医疗政策发布），有新内容时自动通知团队。

**对你的价值**：
- **降低重复劳动**：很多项目中有大量重复性工作（如数据收集、消息转发、报表生成），用工作流工具可以一次配置、长期自动运行。
- **提升用户体验**：让你的应用能够"7×24 小时自动响应"，而不需要人工值守。
- **小白友好**：这类工具通常提供可视化界面，拖拽式配置，不需要写很多代码。



### 推荐工具

#### **n8n**
- **官网**：[https://n8n.io](https://n8n.io)
- **为什么推荐**：
  - 开源且功能强大，支持几百种应用集成（如微信、邮件、数据库、AI 服务等）。
  - 提供可视化工作流编辑器，可以像"连连看"一样把不同服务连接起来。
  - 可以自己部署（需要服务器），也可以使用官方云服务（付费，但有免费试用）。
- **适用场景**：
  - 用户提交表单后，自动调用 AI 生成回复并发送邮件。
  - 定时从数据库提取患者反馈，生成分析报告。
  - 把多个系统的数据同步到统一的数据库。
- **如何上手**：
  1. 访问 [https://n8n.io](https://n8n.io) 了解功能，可以先用云服务试用。
  2. 创建一个工作流，选择"触发器"（例如：收到 Webhook 请求、定时任务）。
  3. 添加"操作节点"（例如：调用 AI API、发送邮件、写入数据库）。
  4. 测试工作流，确认无误后激活，系统会自动按照你的配置运行。

#### **Flowith**
- **官网**：[https://flowith.io/](https://flowith.io/)
- **为什么推荐**：
  - 国产工具，中文界面，上手更快。
  - 专注于"AI + 自动化"场景，内置了很多 AI 相关的模板（例如：智能客服、内容生成）。
- **适用场景**：如果你的项目核心是 AI 驱动（例如：患者问答、智能推荐），Flowith 的模板可以让你快速搭建原型。
- **如何上手**：
  1. 访问 [https://flowith.io/](https://flowith.io/) 注册账号。
  2. 浏览模板库，选择一个接近你需求的模板（例如："AI 客服工作流"）。
  3. 根据你的项目需求调整配置，测试后上线。



### 小白使用建议
- **从模板开始**：不要从零配置工作流，先找一个类似的模板（n8n 和 Flowith 都有模板库），在模板基础上修改会快很多。
- **先做简单场景**：例如："收到表单提交 → 发送一封欢迎邮件"，跑通后再逐步增加复杂度。
- **向 Mentor 求助**：如果需要部署 n8n 到服务器，建议找 Mentor 或技术同学协助，避免在环境配置上卡太久。

## 附录：小白常见问题（FAQ）

**Q1：我完全不会编程，能参与这个项目吗？**  
**A**：可以！很多工具（如 Qoder、Fastgpt、V0.app）都支持"用自然语言描述需求，AI 帮你生成代码"。你可以把自己定位为"产品经理"，专注于理解用户需求、设计功能流程，技术实现交给 AI 和工具。当然，学习一些基础的编程知识会让你和 AI 协作得更顺畅。

**Q2：这些工具都要花钱吗？**  
**A**：大部分工具都有免费版或学生优惠。优先使用社区资源（找 Mentor 申请）和本文档推荐的免费资源（如 GitHub Student Pack、GLM 新用户额度）。如果后续项目需要扩大规模，再考虑付费升级。

**Q3：我该先学哪个工具？**  
**A**：建议按项目需求选择：
- 如果要做"知识问答"，先学 Fastgpt。
- 如果要做"网页或小程序"，先学 Qoder。
- 如果要做"自动化流程"，先学 n8n 或 Flowith。
不要贪多，专注于你当前项目最需要的那 1-2 个工具。

**Q4：遇到问题怎么办？**  
**A**：
1. 先问 AI（如 ChatGPT、Claude、Qoder 内置的 AI）："我遇到了 XXX 错误，怎么解决？"
2. 在项目微信群或 GitHub Issue 里求助，说明你的问题、已经尝试的方法和报错截图。
3. 联系 Mentor 或技术同学，预约一次远程协助。

**Q5：这些工具生成的内容可靠吗？会不会有医疗风险？**  
**A**：AI 工具只是辅助，不能替代专业医疗判断。在项目中：
- 明确告知用户"本系统仅供参考，不构成医疗建议"。
- 涉及关键医疗信息时，务必让专业人士（如医生、药师）审核。
- 遵守项目合规要求，不承诺疗效、不推荐具体药物。



> **最后的话**：工具只是手段，我们的目标是"真正帮助到患者和家庭"。希望这份工具箱能让你少走弯路，把更多时间花在理解用户需求、打磨产品体验上。遇到困难不要气馁，记得向团队和社区求助——我们都在陪你一起走！
