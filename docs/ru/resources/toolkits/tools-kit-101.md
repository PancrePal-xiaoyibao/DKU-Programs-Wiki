## 1. Инструменты технологического стека RAG-продуктов

### Что такое RAG?
RAG (Retrieval-Augmented Generation, генерация с усилением поиска) — это технология, позволяющая ИИ "консультироваться" с конкретными документами перед ответом на вопросы. Проще говоря: вы можете загрузить медицинские документы, руководства для пациентов, рекомендации по лечению и т.д. в систему, чтобы ИИ отвечал на вопросы пользователей на основе этого контента, а не выдумывал.

**Ваша ценность**: Если вашему проекту нужны функции предоставления вопросов-ответов о заболеваниях, запросов о руководствах по лекарствам, консультаций по процессу медицинской помощи для пациентов, инструменты RAG могут помочь вам быстро создать "специализированную базу знаний + клиентскую службу ИИ" без необходимости писать код с нуля.



### Рекомендации сообщества (дружественные для новичков)

#### **Fastgpt**
- **Официальный сайт**: [www.fastgpt.io](https://www.fastgpt.io)
- **Онлайн-опыт**: [https://cloud.fastgpt.io/](https://cloud.fastgpt.io/)
- **Почему рекомендуем новичкам**:
  - Поддерживает китайский и английский интерфейсы, низкий порог входа.
  - Предоставляет официальную SaaS-версию, после регистрации можно напрямую создавать систему вопросов-ответов на базе знаний в браузере с помощью перетаскивания, без необходимости самостоятельно развертывать серверы.
  - Подходит для быстрой проверки идей: например, если вы хотите создать "помощника по часто задаваемым вопросам для пациентов с остеосаркомой", вы можете сначала загрузить соответствующие документы в Fastgpt, протестировать эффект, а затем решить, стоит ли углубляться в разработку.
- **Как начать**:
  1. Посетите [https://cloud.fastgpt.io/](https://cloud.fastgpt.io/) и зарегистрируйте аккаунт.
  2. Создайте "базу знаний" и загрузите свои документы (поддерживаются PDF, Word, Markdown и т.д.).
  3. Создайте "приложение", свяжите базу знаний, после простой настройки можете предоставлять услугу вопросов-ответов внешним пользователям.
  4. Можно сгенерировать ссылку для общего доступа или встроить в вашу веб-страницу/мини-программу.

#### **Dify**
- **Официальный сайт**: [https://dify.ai/](https://dify.ai/)
- **Почему рекомендуем**:
  - Открытый исходный код и мощные функции, поддержка подключения различных LLM (больших языковых моделей).
  - Предоставляет визуальную оркестровку рабочих процессов, подходит для сценариев, требующих более сложной логики (например: сначала запрос базы знаний, затем вызов внешнего API, наконец генерация персонализированного ответа).
  - Активное сообщество, относительно полная документация, легко найти решение при возникновении проблем.
- **Применимые сценарии**: Если вашему проекту нужны "вопросы-ответы о знаниях + вызов внешних данных" (например, запрос базы данных лекарств, информации о больницах), Dify будет более гибким выбором.
- **Как начать**:
  1. Посетите официальный сайт для ознакомления с функциями, можете сначала попробовать официальное демо.
  2. Если нужно самостоятельно развернуть, обратитесь к документации по развертыванию на GitHub (требуются определенные знания о сервере и Docker, рекомендуется обратиться за помощью к Наставнику или техническим коллегам).
  3. Настройте базу знаний, выберите LLM, спроектируйте процесс диалога, протестируйте и опубликуйте.



### Поощряется исследование (продвинутый уровень)
Если вы уже знакомы с базовыми инструментами RAG и хотите попробовать более передовые технологии, обратите внимание на:

- **Ragflow**: RAG на основе графовой базы данных, подходит для обработки сложных реляционных данных (например: связи между заболеваниями-симптомами-лекарствами).
- **RAG на основе сотрудничества множественных агентов**: Позволяет нескольким ИИ сотрудничать для выполнения задач (например: один отвечает за поиск документов, один за резюмирование, один за генерацию рекомендаций).
- **Передовые технологические проекты**:
  - **TrustRAG**: Система RAG, ориентированная на надежность и объяснимость, подходит для сценариев с высокими требованиями к точности, таких как медицина.
  - (Больше инструментов постоянно обновляется, приветствуем делиться своими находками в группе)


# 2. Помощники по программированию с ИИ (Coding Copilot)

### Что такое помощник по программированию с ИИ?
Помощник по программированию с ИИ — это инструмент, который может помочь вам писать код, объяснять код, исправлять ошибки и даже генерировать полные функции на основе описания на естественном языке. Вы можете представить его как "круглосуточного онлайн-репетитора по программированию + партнера по парному программированию".

**Ваша ценность**:
- **Новички в программировании**: Можете описывать потребности на естественном языке (например: "помогите мне написать веб-форму для сбора имени и контактной информации пользователя"), ИИ напрямую сгенерирует код и объяснит назначение каждой строки.
- **Студенты с определенной базой**: Можете использовать ИИ для ускорения разработки, например, быстро создавать фреймворк проекта, автоматически дополнять повторяющийся код, помогать отлаживать сообщения об ошибках.
- **Практическое применение в проектах**: Будь то создание мини-программы, веб-страницы или скрипта обработки данных, помощник по программированию с ИИ может значительно снизить порог разработки.

# 3. Ресурсы LLM Token (квота вызова ИИ)

### Что такое LLM Token?
Когда вы используете сервисы ИИ (такие как GPT, Claude, GLM и т.д.), каждый диалог, каждая генерация контента потребляет "Token" (можно понимать как "квоту слов"). Token обычно требует платной покупки, но как студент и участник благотворительного проекта вы можете получить бесплатные или льготные ресурсы через следующие каналы.

**Ваша ценность**:
- Бесплатное получение квоты вызова ИИ для разработки и тестирования проекта.
- Снижение затрат, позволяющее вам смело пробовать различные функции ИИ, не беспокоясь о том, что "за частое использование придется платить".



### 3.1 Ресурсы сообщества (приоритетная рекомендация)

#### **Подать заявку на квоту сообщества у Наставника**
- **Как подать заявку**: Свяжитесь с вашим проектным Наставником или учителем Тянь, объясните потребности вашего проекта и предполагаемое использование.
- **Почему приоритетная рекомендация**:
  - Квота сообщества обычно больше, подходит для использования в производственной среде (например: после официального запуска вашего приложения реальные пользователи будут продолжать вызывать ИИ).
  - Управляется сообществом единообразно, можно избежать риска злоупотребления или превышения личного аккаунта.
- **Примечания**:
  - При подаче заявки укажите назначение (например: "для вопросов-ответов базы знаний RAG" или "для помощника по программированию с ИИ"), чтобы Наставник мог оценить и распределить.
  - Цените ресурсы, не используйте для личных развлечений или коммерческих целей, не связанных с проектом.

# 4. Инструменты автоматизации рабочих процессов

### Что такое автоматизация рабочих процессов?
Инструменты автоматизации рабочих процессов могут помочь вам связать несколько шагов вместе для реализации "автоматизированных задач без участия человека". Например:
- Когда пользователь отправляет вопрос в форме, автоматически отправить его на обработку ИИ и отправить результат ответа пользователю по электронной почте.
- Периодически извлекать данные из базы данных, генерировать отчеты и отправлять их в группу WeChat.
- Мониторить обновления определенного веб-сайта (например, публикации медицинской политики), автоматически уведомлять команду при появлении нового контента.

**Ваша ценность**:
- **Снижение повторяющегося труда**: В многих проектах есть большое количество повторяющейся работы (например, сбор данных, пересылка сообщений, генерация отчетов), с помощью инструментов рабочих процессов можно один раз настроить и долгосрочно автоматически запускать.
- **Улучшение пользовательского опыта**: Позволяет вашему приложению "автоматически отвечать 7×24 часа" без необходимости ручного дежурства.
- **Дружественно для новичков**: Эти инструменты обычно предоставляют визуальный интерфейс, настройку путем перетаскивания, не требуют написания большого количества кода.



### Рекомендуемые инструменты

  - Предоставляет визуальный редактор рабочих процессов, можно подключать различные сервисы как в "игре в поиск соответствий".

- **Применимые сценарии**:
  - После отправки пользователем формы автоматически вызвать ИИ для генерации ответа и отправки электронного письма.
  - Периодически извлекать отзывы пациентов из базы данных, генерировать аналитический отчет.
  - Синхронизировать данные из нескольких систем в единую базу данных.
- **Как начать**:
  1. Посетите [https://n8n.io](https://n8n.io) для ознакомления с функциями, можете сначала попробовать облачный сервис.
  2. Создайте рабочий процесс, выберите "триггер" (например: получение Webhook-запроса, запланированная задача).
  3. Добавьте "узел операции" (например: вызов ИИ API, отправка электронной почты, запись в базу данных).
  4. Протестируйте рабочий процесс, после подтверждения отсутствия ошибок активируйте, система будет автоматически работать в соответствии с вашей конфигурацией.

#### **Flowith**
- **Официальный сайт**: [https://flowith.io/](https://flowith.io/)
- **Почему рекомендуем**:
  - Отечественный инструмент, китайский интерфейс, быстрее освоить.
  - Фокус на сценариях "ИИ + автоматизация", встроено множество шаблонов, связанных с ИИ (например: интеллектуальная клиентская служба, генерация контента).
- **Применимые сценарии**: Если ядро вашего проекта — это драйвер ИИ (например: вопросы-ответы пациентов, интеллектуальные рекомендации), шаблоны Flowith могут помочь вам быстро создать прототип.
- **Как начать**:
  1. Посетите [https://flowith.io/](https://flowith.io/) для регистрации аккаунта.
  2. Просмотрите библиотеку шаблонов, выберите шаблон, близкий к вашим потребностям (например: "рабочий процесс клиентской службы ИИ").
  3. Настройте конфигурацию в соответствии с потребностями вашего проекта, протестируйте и запустите.



### Рекомендации для новичков

## Приложение: Часто задаваемые вопросы новичков (FAQ)

**Q1: Я совершенно не умею программировать, могу ли я участвовать в этом проекте?**  
**A**: Да! Многие инструменты (такие как Qoder, Fastgpt, V0.app) поддерживают "описание потребностей на естественном языке, ИИ помогает генерировать код". Вы можете позиционировать себя как "продукт-менеджер", сосредоточившись на понимании потребностей пользователей, проектировании функциональных процессов, а техническую реализацию оставить ИИ и инструментам. Конечно, изучение некоторых базовых знаний программирования сделает ваше сотрудничество с ИИ более гладким.

**Q2: Все эти инструменты платные?**  
**A**: Большинство инструментов имеют бесплатную версию или студенческие скидки. Приоритет использования ресурсов сообщества (обратитесь к Наставнику) и рекомендованных в этом документе бесплатных ресурсов (таких как GitHub Student Pack, квота для новых пользователей GLM). Если в дальнейшем проекту нужно расширяться, рассмотрите платное обновление.

**Q3: Какой инструмент мне следует изучить первым?**  
**A**: Рекомендуется выбирать в соответствии с потребностями проекта:
- Если нужно делать "вопросы-ответы о знаниях", сначала изучите Fastgpt.
- Если нужно делать "веб-страницу или мини-программу", сначала изучите Qoder.
- Если нужно делать "автоматизированный процесс", сначала изучите n8n или Flowith.
Не будьте жадными, сосредоточьтесь на 1-2 инструментах, которые наиболее нужны вашему текущему проекту.

**Q4: Что делать при возникновении проблем?**  
**A**:
1. Сначала спросите ИИ (такой как ChatGPT, Claude, встроенный ИИ Qoder): "Я столкнулся с ошибкой XXX, как решить?"
2. Обратитесь за помощью в проектную группу WeChat или GitHub Issue, объясните вашу проблему, уже испробованные методы и скриншот ошибки.
3. Свяжитесь с Наставником или техническим коллегой, договоритесь о сеансе удаленной помощи.

**Q5: Надежен ли контент, генерируемый этими инструментами? Есть ли медицинские риски?**  
**A**: Инструменты ИИ — это только помощь, они не могут заменить профессиональную медицинскую оценку. В проекте:
- Четко информируйте пользователей "эта система только для справки, не является медицинской рекомендацией".
- При работе с ключевой медицинской информацией обязательно привлекайте специалистов (таких как врачи, фармацевты) для проверки.
- Соблюдайте требования соответствия проекта, не обещайте терапевтических эффектов, не рекомендуйте конкретные лекарства.



> **Заключительное слово**: Инструменты — это только средства, наша цель — "действительно помочь пациентам и семьям". Надеемся, что этот набор инструментов поможет вам избежать обходных путей и потратить больше времени на понимание потребностей пользователей и совершенствование пользовательского опыта. Не унывайте при возникновении трудностей, помните обращаться за помощью к команде и сообществу — мы все идем с вами вместе!
